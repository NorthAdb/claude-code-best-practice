## 委托规则

任何更新、修改或修复演示文稿的请求必须由匹配的按演示文稿分配的 Agent 处理。**永远不要直接编辑演示文稿 HTML。** 根据用户所指的演示文稿进行路由：

| 演示文稿 | 路径 | Agent |
|---|---|---|
| Vibe Coding → Agentic Engineering | `presentation/vibe-coding-to-agentic-engineering/index.html` | `presentation-vibe-coding` |
| Claude Code & Gemini CLI（GDG Kolachi 活动演示） | `presentation/2026-04-25-gdg-kolachi-cli-claude-code-gemini/index.html` | `presentation-claude-gemini` |
| Claude Code 最佳实践（规范可复用演示） | `presentation/claude-code-best-practice/index.html` | `presentation-claude-code` |

通过 Agent 工具调用：

```
Agent(subagent_type="presentation-vibe-coding", description="...", prompt="...")
Agent(subagent_type="presentation-claude-gemini", description="...", prompt="...")
Agent(subagent_type="presentation-claude-code", description="...", prompt="...")
```

如果用户说"演示文稿"而没有具体说明是哪个，请先询问他们指的是哪个再进行委托。注意"主演示文稿"或"最佳实践演示"通常指的是 `presentation-claude-code` — 即规范可复用演示 — 但如果存在歧义，请确认。

## 原因

每个演示文稿都有自己的幻灯片编号、层级系统和目标受众。按演示文稿分配的 Agent 让每个演示都能保持一个聚焦的知识库，并自我演进而不会相互污染。vibe-coding Agent 预加载了该演示特有的框架/结构/样式技能。claude-gemini Agent 面向非技术性的 GDG 活动受众，并使用带有 6 个层级的旅程栏。claude-code Agent 拥有规范可复用的最佳实践演示（于 2026-04-30 从 GDG 演示分叉而来）— 相同的受众语气，更简单的结构（仅层级徽章，无旅程栏），与活动无关的身份。
