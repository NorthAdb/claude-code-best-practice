# Claude Code：使用量、速率限制与额外用量

## 概述

使用订阅计划（Pro、Max 5x、Max 20x）的 Claude Code 有使用量限制，这些限制在滚动时间窗口内重置。三个内置斜杠命令可帮助你监控和管理用量：

| 命令 | 描述 | 适用用户 |
|---------|-------------|--------------|
| `/usage` | 查看计划限制和速率限制状态 | Pro、Max 5x、Max 20x |
| `/extra-usage` | 配置达到限制时的按量付费溢出机制 | Pro、Max 5x、Max 20x |
| `/cost` | 显示当前会话的 Token 使用量和花费 | API 密钥用户 |

---

## `/usage` — 查看你的限制

显示当前计划的使用量限制和速率限制状态。用于在达到限制前检查还剩多少容量。

---

## `/extra-usage` — 达到限制后继续工作

`/extra-usage` 命令配置**按量付费溢出计费**，这样当达到计划的速率限制时，Claude Code 可以无缝继续工作，而不是被阻止。

### 工作原理

1. 你达到计划的速率限制（限制每 5 小时重置一次）
2. 如果启用了额外用量且有可用余额，Claude Code 会继续工作而不会中断
3. 溢出 Token 按**标准 API 费率**计费，与订阅费用分开

### 设置方法

CLI 中的 `/extra-usage` 命令将引导你完成配置。你也可以在 claude.ai 的**设置 > 用量**页面中进行配置：

1. 启用额外用量
2. 添加支付方式
3. 设置**月度支出上限**（或选择不设上限）
4. 可选地添加**预付款项**，当余额低于阈值时自动充值

### 关键细节

| 细节 | 值 |
|--------|-------|
| 每日赎回限额 | $2,000/天 |
| 计费 | 与订阅分开，按标准 API 费率 |
| 限制重置周期 | 每 5 小时 |

### 已知问题

截至 2026 年 2 月，`/extra-usage` CLI 命令[缺少文档](https://github.com/anthropics/claude-code/issues/12396)，可能会打开登录窗口但没有清晰的配置选项。目前通过 **claude.ai 网页界面**配置是更可靠的方式。

## `/cost` — 会话花费（API 用户）

对于使用 API 密钥（而非订阅计划）进行身份验证的用户，`/cost` 显示：

- 当前会话的总花费
- API 时长和实际时间
- Token 使用量明细
- 所做的代码更改

此命令不适用于 Pro/Max 订阅用户。

---

## 快速模式和额外用量

快速模式（`/fast`）使用 Claude Opus 4.6 并提供更快的输出。它与额外用量有特殊的计费关系：

- 快速模式的使用量**始终从第一个 Token 开始计入额外用量**
- 即使你的订阅计划中还有剩余用量，此规则依然适用
- 快速模式不会消耗计划中包含的速率限制

这意味着你需要启用并充值额外用量才能使用 `/fast`。

## CLI 启动参数

两个与使用量预算相关的启动参数（仅限 API 密钥用户，print 模式）：

| 参数 | 描述 |
|------|-------------|
| `--max-budget-usd <AMOUNT>` | API 调用的最大美元金额，超出后停止 |
| `--max-turns <NUMBER>` | 限制代理循环次数 |

完整列表请参见 [CLI 启动参数参考](claude-cli-startup-flags.md)。
