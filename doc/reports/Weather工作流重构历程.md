## 2. 新 LEVELS 映射（键名和颜色不变）

新的章节顺序是：**代理 Agent → 技能 Skill → 上下文 Context → CLAUDE.md → 命令 Command**。`workflow` 级别键从 `data-level` 使用中退役（章节分隔符失去 `data-level`）。`LEVELS` 映射仍会携带 `workflow` 用于旅程栏历史显示，但没有幻灯片会触发它。

**修订方案**：由于没有幻灯片携带 `data-level="workflow"`，因此从 `LEVELS` 映射中彻底移除 `workflow` 级别。旅程栏最高到 `commands`（83%）。这没问题——工作流章节作为命令章节内部的高潮呈现，而非独立主题。

实际上旅程栏填充到 83% 而不是 100% 作为结尾章节感觉不够完满。更好的方案：**将命令 + 工作流合并为一个名为"命令与工作流"的章节**，使用 `data-level="commands"`。在 `LEVELS` 中将 `workflow` 级别保留为 100%，并将 `data-level="workflow"` 分配给*原*工作流章节分隔符页——它在命令章节内成为一个视觉过渡。这样旅程栏在工作流幻灯片处填充到 100%。

**最终决定**：在 `LEVELS` 中同时保留 `commands`（83%）和 `workflow`（100%）。将 `data-level="commands"` 分配给命令章节分隔符，将 `data-level="workflow"` 分配给工作流子章节页。旅程刻度保持不变。这与当前结构完全一致——只是内容页重新排序。
