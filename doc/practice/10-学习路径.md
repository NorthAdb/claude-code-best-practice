# Claude Code 学习路径

> 从入门到精通的学习路线图
> 学习顺序：第 10 篇（最终篇）

---

## 学习路线概览

```
第 1 周：概念理解 + 基础配置
第 2 周：Subagent + Skill + Hook 实战
第 3 周：MCP + Agent Teams + 工作流定制
第 4 周+：社区工作流 + 实践出真知
```

---

## 第 1 周：建立基础

### 目标
从零开始，理解 Claude Code 的核心概念，完成基础配置。

### 学习步骤

#### Day 1：阅读概念总览
- 阅读 [01-CONCEPTS-概念总览.md](./01-CONCEPTS-概念总览.md)
- 理解 Subagent、Command、Skill、Hook、MCP 五大概念
- **不用记住所有细节**，重点是知道"有什么"和"干什么用"

#### Day 2：理解子代理
- 阅读 [02-子代理-subagents.md](./02-子代理-subagents.md)
- 重点理解 frontmatter 字段的含义
- 查看官方内置的 5 个 agent 类型

#### Day 3：理解命令和技能
- 阅读 [03-命令-commands.md](./03-命令-commands.md)
- 阅读 [04-技能-skills.md](./04-技能-skills.md)
- 理解 Command vs Skill vs Subagent 的区别

#### Day 4：理解配置体系
- 阅读 [05-设置-settings.md](./05-设置-settings.md)
- 重点理解配置优先级和权限系统
- 查看自己项目的 `.claude/settings.json` 了解当前配置

#### Day 5：第一个实践——运行 `/weather-orchestrator`
```bash
claude
/weather-orchestrator
```
观察 **Command → Agent → Skill** 三级架构如何串联。

#### Day 6：第二个实践——运行 `/powerup`
在 Claude Code 中运行：
```
/powerup
```
从第 1 课开始，过一遍所有 10 个交互课程。

#### Day 7：配置你自己的 CLAUDE.md
```
阅读 07-记忆系统-memory.md
然后帮我的项目创建一份 CLAUDE.md
```

---

## 第 2 周：深入实战

### 目标
构建自己的 Agent 工具箱，配置自动化和权限。

### 学习步骤

#### Day 8：理解 MCP 服务器
- 阅读 [06-MCP服务器-mcp.md](./06-MCP服务器-mcp.md)
- 了解常用的 5 个 MCP 服务器
- 配置至少 1 个 MCP 服务器（推荐 Context7）

#### Day 9：深入理解记忆系统
- 阅读 [07-记忆系统-memory.md](./07-记忆系统-memory.md)
- 理解三层记忆体系
- 查看 `.claude/rules/` 目录了解规则加载

#### Day 10：创建第一个 Subagent
让 Claude 帮你创建一个：
```
帮我创建一个代码审查 Subagent，放在 .claude/agents/code-reviewer.md
要求：使用 sonnet 模型，限制 10 轮对话，只有只读权限
```

#### Day 11：创建第一个 Skill
```
阅读 04-技能-skills.md
帮我创建一个记录我项目 API 设计模式的 Skill
放在 .claude/skills/api-patterns/SKILL.md
```

#### Day 12：创建第一个 Command
```
帮我创建一个 /deploy 命令，执行测试→构建→部署流程
放在 .claude/commands/deploy.md
```

#### Day 13：配置权限系统
- 理解 allow / ask / deny 三种规则
- 使用 `/permissions` 命令管理权限
- 使用 `fewer-permission-prompts` Skill 减少权限提示

#### Day 14：了解 CLI 启动参数
- 阅读 [08-CLI启动参数-cli-startup-flags.md](./08-CLI启动参数-cli-startup-flags.md)
- 学会使用 `-p` 做一次性任务
- 学会使用 `--model` 切换模型

---

## 第 3 周：进阶技巧

### 目标
掌握高级功能，形成自己的开发工作流。

### 学习步骤

#### Day 15-16：Agent Teams
```
帮我启用 Agent Teams，并配置 2 个 team member：
- 一个负责代码审查
- 一个负责测试
```

#### Day 17-18：动态工作流
在提示中加入 `ultracode` 关键词触发动态工作流：
```
ultracode 帮我重构 auth 模块，使用多 agent 并行审查
```

#### Day 19：Scheduled Tasks
```
/loop 1h "检查是否有新的依赖更新"
```

#### Day 20：Remote Control
```
/remote-control
```
设置在 claude.ai 上远程控制你的会话。

#### Day 21：Goal 模式
```
/goal 完成 auth 模块的重构，所有测试通过
```
Claude 会持续工作直到目标达成。

## 第 4 周+：社区与进阶

### 学习目标

| 资源 | 说明 |
|------|------|
| [Superpowers](https://github.com/obra/superpowers) (233k ⭐) | 完整开发工作流 |
| [ECC](https://github.com/affaan-m/everything-claude-code) (218k ⭐) | 全功能扩展套件 |
| [SpecKit](https://github.com/github/spec-kit) (114k ⭐) | 需求到代码的完整工作流 |
| [Matt Pocock Skills](https://github.com/mattpocock/skills) (137k ⭐) | 类型安全优先的 Skill 集合 |
| [gstack](https://github.com/garrytan/gstack) (112k ⭐) | 从规划到部署的完整流程 |

### 建议

1. **先不要急着抄别人的工作流** — 前三周打好基础
2. **选择一个工作流深入** — 推荐从 Superpowers 或 ECC 开始
3. **形成自己的工作流** — 结合个人习惯和项目需求
4. **贡献回社区** — 分享你的最佳实践

## 每日习惯

| 习惯 | 说明 |
|------|------|
| 每天更新 Claude Code | `claude update` |
| 早晨看更新日志 | [Changelog](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) |
| 约 50% 上下文时手动 `/compact` | 比自动 compact 效果更好 |
| 新任务 → 新会话 | 不要在一个会话里混太多任务 |
| 复杂任务先从 Plan 模式开始 | `/plan` 或 `--permission-mode plan` |
| 经常截图分享给 Claude | 遇到问题时直接贴图 |
