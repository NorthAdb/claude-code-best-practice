## 系统概述

天气系统在一个编排工作流中展示了两种不同的Skill模式：
- **Agent Skills（预加载）**：`weather-fetcher` 在启动时注入到 `weather-agent` 中作为领域知识
- **Skills（独立）**：`weather-svg-creator` 由命令通过 Skill 工具直接调用

这展示了**命令 → Agent → Skill** 的架构模式，其中：
- 命令编排工作流并处理用户交互
- Agent 使用其预加载的 Skill 获取数据
- Skill 独立创建视觉输出

## 组件概要

| 组件 | 角色 | 示例 |
|-----------|------|---------|
| **命令** | 入口点，用户交互 | [`/weather-orchestrator`](../.claude/commands/weather-orchestrator.md) |
| **Agent** | 使用预加载 Skill 获取数据（agent skill） | [`weather-agent`](../.claude/agents/weather-agent.md) 预加载了 [`weather-fetcher`](../.claude/skills/weather-fetcher/SKILL.md) |
| **Skill** | 独立创建输出（skill） | [`weather-svg-creator`](../.claude/skills/weather-svg-creator/SKILL.md) |

## 组件详情

### 1. 命令

#### `/weather-orchestrator`（命令）
- **位置**：`.claude/commands/weather-orchestrator.md`
- **目的**：入口点 — 编排工作流并处理用户交互
- **操作**：
  1. 询问用户温度单位偏好（摄氏度/华氏度）
  2. 通过 Agent 工具调用 weather-agent
  3. 通过 Skill 工具调用 weather-svg-creator
- **模型**：haiku

### 2. 带预加载 Skill 的 Agent（Agent Skill）

#### `weather-agent`（Agent）
- **位置**：`.claude/agents/weather-agent.md`
- **目的**：使用其预加载的 Skill 获取天气数据
- **Skills**：`weather-fetcher`（作为领域知识预加载）
- **可用工具**：Read, Skill
- **模型**：sonnet
- **颜色**：green
- **记忆**：project

该 Agent 在启动时将 `weather-fetcher` 预加载到其上下文中。它按照 Skill 的指令获取温度并将值返回给命令。

### 3. Skill

#### `weather-svg-creator`（Skill）
- **位置**：`.claude/skills/weather-svg-creator/SKILL.md`
- **目的**：创建可视化的 SVG 天气卡片并写入输出文件
- **调用方式**：通过命令中的 Skill 工具调用（不预加载到任何 Agent 中）
- **输出**：
  - `orchestration-workflow/weather.svg` — SVG 天气卡片
  - `orchestration-workflow/output.md` — 天气概要

### 4. 预加载 Skill

#### `weather-fetcher`（Skill）
- **位置**：`.claude/skills/weather-fetcher/SKILL.md`
- **目的**：获取实时温度数据的指令
- **数据来源**：阿联酋迪拜的 Open-Meteo API
- **输出**：温度值和单位（摄氏度或华氏度）
- **注意**：这是一个 agent skill — 预加载到 `weather-agent` 中，不直接调用

## 执行流程

1. **用户调用**：用户运行 `/weather-orchestrator` 命令
2. **用户提示**：命令询问用户偏好的温度单位（摄氏度/华氏度）
3. **Agent 调用**：命令通过 Agent 工具调用 `weather-agent`
4. **Skill 执行**（在 Agent 上下文中）：
   - Agent 按照 `weather-fetcher` Skill 的指令从 Open-Meteo 获取温度
   - Agent 将温度值和单位返回给命令
5. **SVG 创建**：命令通过 Skill 工具调用 `weather-svg-creator`
   - Skill 在 `orchestration-workflow/weather.svg` 创建 SVG 天气卡片
   - Skill 将概要写入 `orchestration-workflow/output.md`
6. **结果显示**：向用户展示概要，包括：
   - 请求的温度单位
   - 获取到的温度
   - SVG 卡片位置
   - 输出文件位置

## 关键设计原则

1. **两种Skill模式**：展示了 agent skills（预加载）和 skills（直接调用）两种模式
2. **命令作为编排器**：命令处理用户交互并协调工作流
3. **Agent 用于数据获取**：Agent 使用其预加载的 Skill 获取数据，然后返回
4. **Skill 用于输出**：SVG 创建器独立运行，从命令上下文接收数据
5. **清晰的分离**：获取（Agent）→ 渲染（Skill）— 每个组件有单一职责

## 架构模式

### Agent Skill（预加载）

```yaml
# 在 Agent 定义文件中 (.claude/agents/weather-agent.md)
---
name: weather-agent
skills:
  - weather-fetcher    # 启动时预加载到 Agent 上下文中
---
```

- **Skills 被预加载**：完整的 Skill 内容在启动时注入到 Agent 的上下文中
- **Agent 使用 Skill 知识**：Agent 遵循预加载 Skill 的指令
- **无动态调用**：Skills 作为参考资料，不单独调用

### Skill（直接调用）

```yaml
# 在 Skill 定义文件中 (.claude/skills/weather-svg-creator/SKILL.md)
---
name: weather-svg-creator
description: 创建 SVG 天气卡片...
---
```

- **通过 Skill 工具调用**：命令调用 `Skill(skill: "weather-svg-creator")`
- **独立执行**：在命令的上下文中运行，不在 Agent 内部
- **从上下文接收数据**：使用对话中已有的温度数据
